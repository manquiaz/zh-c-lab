# zh-c-lab

一个用于归档和发布独立 HTML 页面的仓库。

## 项目简介

本仓库旨在收集、整理并托管多个独立的前端 HTML 页面。这些页面通常以单文件形式存在，不依赖复杂后端，适合快速浏览或作为功能示例存档。

仓库本身不针对任何特定站点或域名，仅为各类 HTML 页面提供集中存放与版本管理。

## 目录说明

```
.
├── pages/          # 存放各独立 HTML 页面
├── assets/         # 页面所需的共享资源（如 CSS、JS、图片等）
├── index.html      # 仓库入口页面（可选）
└── README.md       # 本文件
```

- `pages/` 目录下每个子文件夹或文件对应一个独立页面，命名尽量反映内容或功能。
- `assets/` 用于存放多个页面可能复用的公共资源，避免重复。

## 页面归档说明

- 每个页面以完整 HTML 文件形式提交，内部包含所需样式与脚本（或通过相对路径引用 `assets/` 下资源）。
- 页面之间相互独立，无强制依赖关系。
- 归档的页面可能包括：工具类页面、演示页面、实验性功能页面、信息展示页等。

## 维护说明

- 欢迎提交新页面或改进现有页面，请遵循以下原则：
  - 保持页面自包含，尽量减少外部依赖。
  - 在 `pages/` 下创建对应目录或文件，并在 `assets/` 中添加公共资源时注意命名规范。
  - 提交前请确保页面在主流浏览器中基本可用。
- 本仓库不保证所有页面持续更新或兼容最新技术标准，仅供存档与参考。
- 如需删除或修改已有页面，请通过 Pull Request 或 Issue 讨论。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 直接在浏览器中打开 `pages/` 下的 HTML 文件，或通过本地服务器预览（推荐）。
3. 也可通过 GitHub Pages 等静态托管服务部署后在线浏览（需自行配置）。
